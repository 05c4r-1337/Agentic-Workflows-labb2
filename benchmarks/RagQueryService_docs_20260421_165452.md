# IRagQueryService

Defines the contract for services responsible for executing Retrieval-Augmented Generation (RAG) queries. This service handles the entire process from embedding the user query to retrieving context from various data sources and generating a final, grounded answer using a large language model (LLM).

## QueryAsync

Executes a comprehensive RAG query. This method takes a user request, performs semantic search across specified knowledge bases (collections), aggregates the results into a context, and uses an external LLM (via Ollama) to generate a final, grounded answer based *only* on that context.

**Behavioral Constraints:**
The underlying Large Language Model (LLM) is constrained by a strict system prompt that mandates the following behaviors:
1. The answer must be based **ONLY** on the provided context.
2. If the context does not contain the answer, the model must respond with the specific phrase: "I don't have that information in my knowledge base".
3. The response must be concise, helpful, and professional.

| Return Type | Description |

If any internal component fails (e.g., network error, embedding failure), the response will contain a generic error message and the query time recorded up to the point of failure. |
