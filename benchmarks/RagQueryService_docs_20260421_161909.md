### QueryAsync(RagQueryRequest request)

Executes the full RAG workflow. It first generates an embedding for the user query. Then, it iterates through all specified collections, performing semantic searches against Weaviate to gather relevant documents. These results are sorted and filtered. The retrieved content is compiled into a comprehensive context string. Finally, this context, along with the original query, is passed to the configured LLM via `OllamaClient` to generate a final, grounded answer.

**System Prompt and Constraints:**
The LLM is instructed to act as a helpful eCommerce assistant for ViskAnna. The system prompt strictly mandates that the model must:
1. Answer based ONLY on the context provided.
2. If information is missing, state: "I don't have that information in my knowledge base".
3. Be concise and helpful.
4. Include relevant details (e.g., purchase history for customers, step-by-step instructions for navigation).

**Failure Modes:**
*   **Embedding Failure:** If `_embeddingService.GetEmbeddingAsync` fails, the process will catch the exception, log it, and return a generic error message in `RagQueryResponse.Answer`.
*   **Weaviate Failure:** If any `_weaviateService.SemanticSearchAsync` call fails, the exception will be caught, logged, and the process will return a generic error message.
*   **Ollama Failure:** If `_ollamaClient.ChatAsync` fails, the exception will be caught, logged, and the process will return a generic error message.
