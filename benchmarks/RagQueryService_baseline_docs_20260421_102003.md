## IRagQueryService

Defines the contract for a service responsible for executing Retrieval-Augmented Generation (RAG) queries. This interface abstracts the complex process of retrieving relevant information from a knowledge base and using it to generate a coherent answer via a Large Language Model (LLM).

The retrieved results are compiled into a context, which is then passed to the LLM (via Ollama) along with system instructions to generate a final, context-grounded answer.

### QueryAsync(RagQueryRequest request)

Executes the core RAG logic:
1. Measures the elapsed time for the operation.
2. Generates a vector embedding for the user's query.
3. Iterates through all specified collections, performing a semantic search against Weaviate for each.
4. Aggregates and sorts all results based on distance (relevance).
5. If `IncludeSources` is true, it formats the retrieved documents into `SourceDocument` objects.
6. Calls `BuildContext` to structure the retrieved content into a comprehensive context string.
7. Formats a `OllamaChatRequest` using a predefined `SystemPromptTemplate` that guides the LLM's behavior, injecting the context and the original query.
8. Sends the request to the `OllamaClient` and extracts the answer.
9. Records the total query time and returns the final `RagQueryResponse`.
