### Interface: `IRagQueryService`

Defines the contract for services responsible for executing Retrieval-Augmented Generation (RAG) queries. This service handles embedding generation, semantic search across multiple data sources (collections), and synthesizing the final answer using a Large Language Model (LLM).

#### Method: `QueryAsync`

Executes the RAG workflow: generating embeddings from the user query, performing semantic searches against specified knowledge collections, consolidating the retrieved context, and prompting an LLM to generate a final answer based *only* on that context.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `request` | `RagQueryRequest` | The request object containing the user's query, the collections to search, and configuration options like maximum results. |

| Return Type | Description |
| :--- | :--- |
| `Task<RagQueryResponse>` | A task that resolves to a response object containing the generated answer, the model used, and optionally, a list of source documents used to formulate the answer. |

**Code Example:**

```csharp
// Assuming 'service' is an instance of IRagQueryService
var request = new RagQueryRequest 
{ 
    Query = "What is my last order status?", 
    Collections = new List<string> { "Customers", "Orders" }, 
    MaxResults = 5, 
    IncludeSources = true 
};

var response = await service.QueryAsync(request);

// Output:
// response.Answer will contain the LLM's synthesized response.
// response.Sources will contain SourceDocument objects detailing where the information came from.
// response.QueryTimeMs will contain the total time taken in milliseconds.
```
